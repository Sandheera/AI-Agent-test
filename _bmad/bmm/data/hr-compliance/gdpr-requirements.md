# GDPR Data Privacy Requirements

## Overview
The General Data Protection Regulation (GDPR) applies to all organizations processing personal data of EU residents, regardless of where the organization is located.

## Key Principles

### 1. Lawfulness, Fairness, and Transparency
- Processing must be transparent to the data subject
- Must have one of six lawful bases for processing
- Privacy notices must be provided clearly
- Opt-out mechanisms for certain processing

### 2. Purpose Limitation
- Specify purpose for data collection upfront
- Only process for stated purpose
- New purpose requires new legal basis
- Document purpose in privacy policy

### 3. Data Minimization
- Collect only necessary personal data
- Regular review to delete unnecessary data
- Implement data retention limits
- Do not retain data indefinitely

### 4. Accuracy
- Keep personal data accurate and current
- Allow individuals to correct data
- Remove inaccurate data
- Maintain correction requests log

### 5. Integrity and Confidentiality
- Protect personal data with appropriate security
- Prevent unauthorized access, alteration, loss
- Implement encryption for sensitive data
- Maintain audit logs of access

### 6. Accountability
- Demonstrate compliance through documentation
- Conduct Data Protection Impact Assessments (DPIA)
- Maintain processing records
- Regular compliance audits

## Employee Rights Under GDPR

### Right to Access (Article 15)
```
Request: Employee can request copy of personal data
Timeline: 30 days (extendable by 60 days)
Format: Copy in commonly used format
Frequency: One free per year, others may be charged
Action: Provide all personal data held
```

### Right to Rectification (Article 16)
```
Request: Employee can correct inaccurate data
Timeline: 30 days to correct
Scope: All inaccurate personal information
Documentation: Keep records of corrections
Cost: Free to employee
```

### Right to Erasure (Article 17)
```
Request: "Right to be forgotten"
Valid When:
  - Data no longer necessary for purpose
  - Consent withdrawn
  - Processing unlawful
  - Legal obligation to erase
  - Special category data breach

Exceptions:
  - Legal obligations (tax, labor law)
  - Legal claims
  - Public interest
  - Public health interest

Timeline: 30 days
Cost: Free
```

### Right to Restrict Processing (Article 18)
```
Request: Limit how personal data is used
Valid When:
  - Accuracy disputed
  - Processing unlawful
  - Data no longer needed (but legal hold)
  - Object to processing (pending decision)

Effect:
  - Can only store data
  - Cannot otherwise process
  - Exception for legal claims
  - Notify data subject before lifting restriction
```

### Right to Data Portability (Article 20)
```
Request: Export personal data in machine-readable format
Valid For:
  - Any data provided by individual
  - Any data generated through activity

Format: Common machine-readable format (CSV, JSON, XML)
Scope: All personal data held
Timeline: 30 days
Cost: Free
```

### Right to Object (Article 21)
```
Request: Opt-out from processing for certain purposes
Valid For:
  - Direct marketing
  - Profiling
  - Legitimate interests processing

Effect: Cease processing (unless compelling interest)
Timeline: Respond within 30 days
Cost: Free
```

### Rights Related to Automated Decisions (Article 22)
```
Prohibition: No solely automated profiling with legal effect
Exceptions:
  - Necessary for contract
  - Authorized by law
  - Explicit consent

Right to: Human review of significant decisions
```

## Employee Data Processing in HR

### Recruitment Data
```
Lawful Basis: Legitimate interests / Contract preparation
Retention: 6 months after hiring decision
Processing: Resume screening, evaluation, communication
Rights: Access, rectification, erasure, portability
```

### Employment Records
```
Lawful Basis: Contract / Legal obligation
Retention: Duration + 3-5 years (varies by document type)
Processing: Salary, benefits, performance, discipline
Rights: Limited by labor law requirements
```

### Health and Safety Records
```
Lawful Basis: Legal obligation / Vital interests
Retention: Duration + 10 years
Processing: Injury reporting, vaccination status, accommodations
Special: Special category data - extra protections required
Rights: Limited by occupational health requirements
```

### Performance and Discipline
```
Lawful Basis: Contract / Legitimate interests
Retention: Duration + 2 years
Processing: Reviews, improvement plans, discipline
Rights: Access to records
Privacy: Cannot access without consent (limited exceptions)
```

### Monitoring and Surveillance
```
Requirements:
  - Clear notice before monitoring
  - Legitimate business purpose
  - Proportionate to purpose
  - Regular necessity assessment

Types and Rules:
  - Email monitoring: Permissible with notice
  - Video surveillance: Limited to security purposes
  - Location tracking: Only essential positions
  - Keystroke monitoring: Highly restricted
```

## Data Protection Officer (DPO)

### When DPO is Required
- Public authorities (most cases)
- Large-scale systematic monitoring
- Large-scale processing of special categories of data

### DPO Responsibilities
- Monitor compliance with GDPR
- Cooperate with supervisory authority
- Be point of contact for authorities
- Conduct data protection impact assessments
- Advise on obligations and requirements
- Maintain records of processing activities

## Data Protection Impact Assessment (DPIA)

### Required When Processing:
- Special category data at scale
- Systematic monitoring at scale
- Automated decision-making with legal effect
- Processing could result in high risk to rights

### Contents:
- Description of processing
- Assessment of necessity and proportionality
- Risk assessment to rights and freedoms
- Mitigation measures proposed
- Safeguards implemented

## Data Breach Response

### Detection and Assessment (Article 33-34)
```
Timing: "Without undue delay" (typically 24-48 hours)
Investigation: Determine scope and impact
Notification: Notify supervisory authority within 72 hours (unless low risk)
Records: Maintain breach register
```

### Notification to Supervisory Authority
```
Required When: High risk to rights
Deadline: 72 hours from discovery
Contents:
  - Description of breach
  - Likely consequences
  - Measures taken/proposed
  - DPO or contact point

Exceptions: Unlikely to pose risk
```

### Notification to Data Subjects
```
Required When: High risk to individual
Timeline: Without undue delay
Method: Clear, intelligible language
Contents:
  - Description of breach
  - Likely consequences
  - Measures to protect data
  - Contact point for more information

Exceptions: Unlikely to result in high risk
```

## Data Sharing and Transfers

### Third-Party Data Processing
```
Requirement: Data Processing Agreement (DPA)
Contents:
  - Processor only processes on instructions
  - Confidentiality clauses
  - Security measures
  - Subprocessor authorization
  - Data subject rights enforcement
  - Deletion/return of data
  - Assistance with data subject rights
  - Audit/inspection rights
```

### International Data Transfers
```
EU to Non-EU: Requires transfer mechanism
Mechanisms:
  - Adequacy Decision (limited countries)
  - Standard Contractual Clauses (SCCs)
  - Binding Corporate Rules (BCRs)
  - Other legal mechanisms

Additional Safeguards Required:
  - Supplementary measures
  - Encryption
  - Access controls
  - Regular assessment
```

## Compliance Checklist

- [ ] Identify lawful basis for all processing
- [ ] Document processing in ROPA (Records of Processing Activities)
- [ ] Update privacy notices with new language
- [ ] Implement data subject rights processes
- [ ] Establish data retention schedules
- [ ] Implement data security measures
- [ ] Appoint DPO if required
- [ ] Conduct DPIAs for high-risk processing
- [ ] Execute Data Processing Agreements with vendors
- [ ] Train employees on data protection
- [ ] Establish breach response procedures
- [ ] Regular compliance audits
- [ ] Document consent where required
- [ ] Implement access controls
- [ ] Enable data portability format exports
